گزارش تحلیل فنی سیستم VirtinSaz (ورتبن‌ساز)
تاریخ گزارش: ۱۸ بهمن ۱۴۰۳

۱. هدف سیستم:
پلتفرم ساخت منوی دیجیتال و وب‌سایت‌های تک‌صفحه‌ای برای کسب‌وکارهای خدماتی (رستوران، کافه و ...).

۲. تکنولوژی‌های مورد استفاده:
- بک‌اِند: Python / Django REST Framework
- فرانت‌اِند: Next.js (App Router) / Tailwind CSS / Shadcn UI
- مدیریت وضعیت: React Hooks & Context API

۳. وضعیت بخش‌های آماده (Ready Features):
- احراز هویت: کامل (OTP-based)
- داشبورد مدیریتی: کامل (نمایش آمار، وضعیت اشتراک، دسترسی سریع)
- مدیریت منو: کامل (ایجاد دسته‌بندی و محصول با جزئیات کامل)
- سیستم QR Code: کامل (تولید خودکار و امکان دانلود)
- سیستم قالب‌ها: ۳ قالب (Minimal, Traditional, Modern) با قابلیت پیش‌نمایش آنی
- تنظیمات سایت: امکان تغییر لوگو، کاور و تنظیمات JSON-based

۴. مدل‌ها و ساختار دیتابیس (Models):

الف) بخش کاربران (Accounts):
- User: شماره موبایل (کلید اصلی)، نام رستوران، وضعیت فعال بودن، آواتار.
- OTP: کد تایید، شماره موبایل، وضعیت استفاده.

ب) بخش سایت (Sites):
- SiteCategory: نام و اسلاگ دسته‌بندی (مثلاً: رستوران سنتی).
- Theme: نام قالب، شناسه برای فرانت‌اِند، تصویر پیش‌نمایش، تنظیمات پیش‌فرض (JSON).
- Site: صاحب سایت، نام، اسلاگ (URL اختصاصی)، لوگو، کاور، تنظیمات ظاهری.

ج) بخش منو (Menu):
- ProductCategory: نام، ترتیب نمایش، وضعیت فعالی.
- Product: عنوان، توضیح، قیمت، درصد تخفیف، تصویر، وضعیت موجودی، محبوبیت.
- ProductTag: نام و رنگ برچسب (مثلاً: تند).

۵. جریان کاری سیستم (Workflow):
- کاربر وارد شده و در صورت نداشتن سایت، وارد جریان CreateSiteFlow می‌شود.
- پس از ساخت سایت، دسترسی به مدیریت محصولات باز می‌شود.
- هر محصول در دیتابیس به سایتِ کاربر متصل (Link) است.
- مشتری با آدرس /preview/[slug] منو را با تم انتخاب شده مشاهده می‌کند.
- آمار بازدیدها و محصولات محبوب به صورت خودکار در داشبورد بروزرسانی می‌شوند.
